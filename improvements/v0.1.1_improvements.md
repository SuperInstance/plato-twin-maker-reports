# v0.1.1 Improvements — Hotfix & Stability

**Focus:** Fix confirmed bugs and harden the core pipeline.

---

## 1. Fix Double-Arrow Signatures in Python Extraction

The regex at line 241 does not correctly handle complex return type annotations containing colons (e.g., `Callable[[int], str]`). This produces malformed signatures like `def foo() -> -> Callable[[int]`.

**Fix:** Replace the regex with proper balanced-paren matching or use a two-pass approach:
1. Match `def name(args)` with the opening `(` and matching `)`
2. Everything between the closing `)` and the final unescaped `:` is the return annotation

**See:** [Bug 1: Double-Arrow Signatures](../bugs/01_double_arrow_signature.md)

---

## 2. Fix Async Function Extraction

Both Python and JavaScript extractors miss async functions. Python only matches `def `, not `async def `. JavaScript only matches named functions, not arrow functions or async arrow assignments.

**Fix:** 
- Python: Change `if stripped.startswith('def ')` to check for `async def ` as well. Update the regex to `r'(?:async\s+)?def\s+(\w+)\s*\((.*?)\)(.*?):'`
- JavaScript: Add patterns for `const foo = async () => { ... }` and `const foo = async function() { ... }`

**See:** [Bug 3: Missing Async Extraction](../bugs/03_async_extraction.md)

---

## 3. Fix Language Detection Weights

The simple extension frequency count misclassifies repos with mixed languages, especially C++/C and TypeScript/JavaScript.

**Fix:** Implement a weighted scoring system where source-code extensions (`.py`, `.cpp`, `.ts`) get 3× the weight of config files (`.json`, `.yaml`, `.toml`, `.md`). Additionally, detect C++ context for `.h` files when `.cpp` or `.cc` files are present.

**See:** [Bug 2: Language Detection Misclassification](../bugs/02_language_detection.md)

---

## 4. Improve Error Handling in Module Extraction

Currently, if any source file causes an exception during extraction, the entire twin creation fails. This is fragile for repos with non-standard file encoding, binary files misnamed with code extensions, or corrupted files.

**Fix:** Wrap each file extraction in a try/except block:

```python
def _extract_modules(self):
    for f in self.path.rglob('*'):
        if not f.is_file():
            continue
        ext = f.suffix.lower()
        if ext not in SUPPORTED_LANGUAGES:
            continue
        try:
            content = f.read_text(encoding='utf-8', errors='replace')
            # ... dispatch to language extractor ...
        except Exception as e:
            print(f"[ptwin] Warning: skipping {f}: {e}")
            continue
```

Log warnings for skipped files so users can investigate, but the twin creation process continues.

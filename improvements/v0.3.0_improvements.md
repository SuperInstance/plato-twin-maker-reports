## 9. Batch Multi-Repo Processing

Accept a list of repositories for batch twin creation:

```bash

python -m plato_twin_maker --batch repos.txt

# From stdin
echo "repo1\nrepo2\nrepo3" | python -m plato_twin_maker --batch -

# Glob pattern
python -m plato_twin_maker --batch "repos/*/"
```

## 10. Streaming Tile Submission

Currently, all tiles are submitted to PLATO at once at the end of `make()`. For large repos, this means:
- All tiles must fit in memory
- If the process crashes mid-submission, zero tiles were submitted
- No progress feedback during submission

**Fix:** Submit tiles incrementally as they're created:

```python
def _submit_tile(self, tile: TileContent) -> bool:
    """Submit a single tile to PLATO immediately."""
    # POST tile.content to PLATO room
    # Returns True if accepted, False if retry needed

def make(self) -> PlatoTwin:
    """Create twin with streaming PLATO submission."""
    for mod in self.analysis.modules:
        tile = self.creator._module_to_tile(mod)
        self.twin.tiles.append(tile)
        if not self.dry_run:
            self._submit_tile(tile)  # Submit immediately
            print(f"  ✓ {tile.source_module}")
```

This also enables graceful degradation: if PLATO becomes unavailable mid-run, tiles submitted so far are preserved.

## 11. Configurable Confidence Thresholds

Add a `--min-confidence` flag to filter out low-quality extractions:

```bash
# Only include modules with confidence >= 0.8
python -m plato_twin_maker --repo ./project --min-confidence 0.8

# Include everything (default behavior)
python -m plato_twin_maker --repo ./project --min-confidence 0.0
```

The confidence scoring should also be improved:

| Condition | Base Confidence |
|-----------|----------------|
| Has docstring | 0.90 |
| No docstring, has type hints | 0.80 |
| No docstring, no type hints | 0.60 |
| Has both docstring + type hints | 0.95 |
| Has tests (unit tests reference this module) | +0.05 |

```python
def _compute_confidence(self, mod: ModuleInfo) -> float:
    """Compute confidence score for a module extraction."""
    confidence = 0.75
    if mod.docstring:
        confidence += 0.15
    if '->' in mod.signature:  # has return type
        confidence += 0.05
    if len(mod.test_hints) > 0:
        confidence += 0.05
    return min(confidence, 1.0)
```

## 12. CLI Refinements

Add production-quality CLI options:

| Flag | Type | Description |
|------|------|-------------|
| `--output-dir` | Path | Directory for export files (JSON, YAML, MD manifests) |
| `--tags` | String | Additional tags to add to all tiles |
| `--exclude` | Glob | File patterns to exclude (e.g. `**/test_*`, `*.generated.*`) |
| `--max-files` | Int | Maximum files to process (skip large repos) |
| `--quiet` | Flag | Suppress non-error output |
| `--progress` | Flag | Show progress bar |
| `--timeout` | Int | PLATO submission timeout in seconds |
| `--retry` | Int | Maximum PLATO submission retries |
| `--verify` | Flag | Run tests after twin creation and report results |

Example:
```bash
python -m plato_twin_maker \
  --repo ./my-large-project \
  --exclude "**/node_modules" "**/test_*" \
  --max-files 500 \
  --min-confidence 0.8 \
  --tags "production" "legacy" \
  --output-dir ./twin-output \
  --progress \
  --verify
```
